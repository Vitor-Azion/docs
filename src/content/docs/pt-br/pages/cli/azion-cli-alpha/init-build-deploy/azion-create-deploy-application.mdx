Veja os comandos que permitem a inicialização, construção e implantação de edge applications na plataforma da Azion.

---

## Init

O processo de inicialização cria o ambiente necessário para a construção do sua edge application.

### Uso

```sh
  azion init
```

### Saída 

Depois de inicializar uma application, a lista de modos disponíveis será exibida. Em seguida, é necessário escolher um deles.

```sh 
  Getting modes available? Choose a template for your project: (Use arrow keys)
❯ Angular 
  Astro 
  Hexo 
  Next 
  React 
  Vue 
  Vite 
```

Selecione o modo: 

```sh
Choose a mode:  [Use arrows to move, type to filter]
  Javascript (Compute)
  Typescript (Compute)
  Angular (Deliver)
  Astro (Deliver)
  Hexo (Deliver)
  Next (Deliver)
  React (Deliver) 
```


### Flags opcionais

#### name

A opção `--name` pode ser especificada, mas não é obrigatória. Se não for informado, o nome do seu projeto será gerado automaticamente.

#### mode

A opção `--mode` pode ser especificada, mas não é obrigatória. Os valores esperados são `deliver` e `compute`.

#### template

A opção `--template` pode ser especificada, mas não é obrigatória. Os valores esperados são:

- Html
- Javascript
- Typescript
- Angular
- Astro
- Hexo
- Next 
- React
- Vue
- static

#### help

A opção `--help` exibe mais informações sobre o comando `build`.

:::note 
O comando `init` possui uma jornada interativa. Ele perguntará se você deseja inicializar um servidor de desenvolvimento local e implantar o projeto.
:::

## build

```bash
  azion build
```

### Flags opcionais

#### help

A opção `--help` exibe mais informações sobre o comando `build`.

#### mode

A opção `--mode` pode ser especificada no caso de ser necessário mudar o modo durante o processo de construção, mas não é obrigatória. Os valores esperados são `deliver` ou `compute`.

#### preset

A opção `--preset` pode ser especificada no caso de ser necessário alterar o preset durante o processo de criação, mas não é obrigatória. Os valores esperados são:

- Html
- Javascript
- Typescript
- Angular
- Astro
- Hexo
- Next 
- React
- Vue
- static

## deploy 

O comando `deploy` implanta sua application na Plataforma Edge da Azion.

### Uso

```bash
  azion deploy
```

### Flags opcionais

#### path 

A opção `--path` é usada quando é necessário informar o caminho do arquivo para a aplicação estática que está sendo implantada.

#### help

A opção `--help` exibe mais informações sobre o comando `deploy`.

---

## Flags herdadas

Existem algumas opções globais que são herdadas pelos comandos `init`, `build` e `deploy`. Confira-as:

```sh
-c, --config string      Define a pasta de configuração da Azion apenas para o comando atual, sem alterar as configurações persistentes.
-d, --debug              Exibe logs em nível de depuração
-l, --log-level string   Exibe logs em nível de depuração (padrão "info")
-s, --silent             Silencia completamente os logs; geralmente usado para fins de automação
-t, --token string       Salva um personal token fornecido localmente para autorizar comandos do CLI
-y, --yes                Responde automaticamente a todas as interações sim/não com sim
```
